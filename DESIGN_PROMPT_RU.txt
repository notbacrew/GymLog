# PROMPT: Воссоздай дизайн приложения GymLog 1:1 (iOS, SwiftUI)

Цель: Создать макет приложения, визуально и поведенчески идентичный приложению GymLog. Ниже — исчерпывающая спецификация дизайна, компонентов, типографики, цветов, отступов, состояний, и экранов. Следуй ей строго.

1) Базовые принципы
- Платформа: iOS 15+.
- Фреймворк UI: SwiftUI. Визуальный стиль должен соответствовать нативным паттернам iOS.
- Темы: системная, светлая, темная. Переключение тем — мгновенное, без перезагрузки экранов.
- Сеточная система и отступы: основной отступ 16pt, малый 8pt. Вертикальный ритм кратен 8pt. Внешние поля контейнеров по краям — 16pt. Между элементами списков — 8pt.
- Скругления: карточки 12pt, интерактивные элементы/поля 8pt. Картинки в списках — с углом 8pt, если они прямоугольные.
- Высоты элементов: стандартные кнопки — 44pt. Таббар — системная высота iOS.
- Иконки: SF Symbols, размеры 16–24pt, цвет иконок соответствует контексту (primary/secondary/semantic).
- Тени карточек: мягкие, невысокая высота, размытость ~8–12, непрозрачность ~10–15%.

2) Цвета (семантическая палитра)
- Primary: синий #007AFF (iOS системный синий). Используется для акцентов, активных состояний, кнопок «действие».
- Secondary: серый #8E8E93 для вторичного текста/иконок.
- Success: зеленый #34C759 (подтверждения/позитивные статусы).
- Warning: оранжевый #FF9500 (предупреждения/нейтрально-важные акценты).
- Danger: красный #FF3B30 (ошибки/удаление).
- Категории упражнений:
  - Грудь: красный (base: #FF3B30)
  - Спина: синий (base: #007AFF)
  - Ноги: зеленый (base: #34C759)
  - Плечи: оранжевый (base: #FF9500)
  - Руки: фиолетовый (base: #AF52DE)
  - Пресс: желтый (base: #FFCC00)
  - Кардио: розовый (base: #FF2D55)
- Текст в светлой теме: основной — near-black #111111, вторичный — #6C6C70. В темной теме: основной — #FFFFFF, вторичный — #9A9AA0.
- Разделители и бордеры: тонкие, 1px, адаптивный системный separator.

3) Типографика (SF Pro)
- Заголовки H1: 28–32pt, Bold.
- Подзаголовки H2: 20pt, Semibold.
- Разделы/карточки заголовки H3: 17pt, Semibold.
- Основной текст: 16–17pt, Regular.
- Вторичный текст/описания: 14–15pt, Regular.
- Микротекст/подписи: 12–13pt, Regular.
- Межстрочные интервалы — системные, без искусственного уплотнения.

4) Общие компоненты
- Карточка: контейнер со скруглением 12pt, светлый фон (light: #FFFFFF, dark: #1C1C1E), тень лёгкая. Внутренние отступы: 16pt. Заголовок (H3) сверху, контент ниже. Разделители внутри — тонкие (если нужно разделение строк).
- Кнопка (primary): высота 44pt, скругление 8pt, фон — Primary, текст белый. Состояния: normal (100%), pressed (opacity 0.85 и лёгкое scale 0.98), disabled (opacity 0.5, события отключены).
- Кнопка (secondary/outline): высота 44pt, скругление 8pt, бордер 1px системный, текст — Primary, фон — clear. Pressed: фон с лёгкой заливкой primary 10%.
- Поле ввода: высота ~44pt, скругление 8pt, бордер тонкий (separator), внутренние отступы горизонтальные 12–16pt. Placeholder — вторичный цвет.
- Теги/бейджи: скругление капсульное, фон — категория с opacity 0.15, текст — категория base, размер текста 13–14pt, внутренние отступы вертикальные 4pt, горизонтальные 8–10pt.
- Пустые состояния (empty state): иконка SF 40–64pt, вторичный текст с подсказкой, кнопка действия ниже.
- Ошибки/инфо-баннеры: фон Danger/Warning с низкой непрозрачностью (10–15%), текст соответствующего цвета, скругление 8pt, иконка слева 20–24pt.

5) Навигация и иерархия экранов
- Главный контейнер: TabView (нижняя навигация) с 7 вкладками в порядке:
  1. Главная — «house.fill», заголовок «Главная»
  2. Тренировки — «dumbbell.fill», «Тренировки»
  3. Упражнения — «list.bullet», «Упражнения»
  4. Прогресс — «chart.line.uptrend.xyaxis», «Прогресс»
  5. Достижения — «trophy.fill», «Достижения»
  6. Таймер — «timer», «Таймер»
  7. Профиль — «person.circle», «Профиль»
- Выбранная вкладка окрашивается в Primary, невыбранные — secondary. Подписи табов всегда видимы.
- Заголовки экранов — системные NavigationTitle (кроме специфических карточных компоновок).

6) Экраны (детальная раскладка)

6.1) Экран «Главная» (Home)
- Верхний приветственный блок: «Привет, <имя>» (H2), под ним короткий вторичный текст.
- Блок «Быстрые действия»: горизонтальный стек кнопок/плиток (два элемента по 1/2 ширины), каждая — карточка 12pt, иконка (24pt) слева, заголовок 16–17pt справа. Действия: «Новая тренировка», «Новое упражнение».
- Статистика недели: карточка с 3–4 метриками (например, тренировки, подходы, общий вес). Каждая метрика — вертикальная мини-плитка: крупное число (H2) + подпись (12–13pt, вторичный).
- Секция «Сегодня» (если есть тренировка): список коротких карточек тренировки (дата, количество упражнений, суммарный объём).
- «Последние тренировки»: вертикальный список карточек. Элемент списка: слева дата (иконка «calendar» 16–20pt), справа заголовок/заметки (одно-два поля), снизу строка метрик (подходы/повторения/вес). Внутренний вертикальный отступ между строками 4–6pt.
- Карточка «Достижения» (тизер): компактная плашка с количеством разблокированных/прогресса.

6.2) «Тренировки» (WorkoutList)
- Верх: поиск (SearchField) + фильтр периода (сегментированный контроль с опциями: Все, Сегодня, Неделя, Месяц, Год). Сегменты — стандартные iOS.
- Список: LazyVStack, элементы — карточки 12pt. В карточке: дата вверху (формат medium), примечание (если есть), строка метрик (подходы, повторения, суммарный вес), бейджи по категориям, если уместно.
- Swipe-to-delete вправо с красной кнопкой «Удалить» (иконка trash), подтверждение через системный alert.
- Pull-to-refresh — системный.
- FAB или кнопка «+» в навбаре для добавления тренировки (иконка «plus»). Экран добавления — модальный лист с большим заголовком.

6.3) «Добавить тренировку» (AddWorkout)
- Верх: дата (DatePicker) в инлайн-стиле.
- Блок «Упражнения»: список добавленных элементов (изначально пуст). Кнопка «Добавить упражнение» (primary) внутри секции.
- Элемент упражнения в тренировки: слева мини-иконка категории (цвет категории), справа название, ниже — строки ввода: подходы (Stepper), повторения (Stepper), вес (TextField с единицами «кг»). Выравнивание по сетке, подписи слева 14–15pt вторичный, значения справа.
- Поле заметок: многострочное, высота по контенту, ограничение 500 символов (визуально показывать остаток при 450+ символах, цвет предупреждения Warning при превышении лимита).
- Кнопка «Сохранить» (primary, фиксируется внизу при длинном контенте). Disabled, если нет ни одного упражнения.

6.4) «Детали тренировки» (WorkoutDetail)
- Заголовок: дата (формат full/medium), подзаголовок — заметка (если есть).
- Секция «Состав»: список упражнений. Каждый элемент: мини-изображение/иконка категории (24–32pt), название, под ним — «подходы × повторения, вес кг». Доп. комментарий, если есть.
- Итоги: карточка с суммарными метриками (количество упражнений, подходов, повторений, общий вес).
- Кнопки: «Редактировать», «Удалить» (вторичная/опасная). Удаление — через confirm sheet/alert.

6.5) «Редактировать тренировку» (EditWorkout)
- Макет идентичен AddWorkout, предзаполненные поля, кнопка «Сохранить изменения».

6.6) «Фильтры тренировок» (WorkoutFilters)
- Сегменты периода (как в списке), выбор категорий чекбоксами/тогглами (цвета категорий в капсульных тегах), кнопка «Применить».

6.7) «Упражнения» (ExerciseList)
- Верх: поиск, фильтрация по категориям в виде горизонтального Scroll сегментов/капсул: «Все» + перечисленные категории. Активная капсула имеет фон категории с opacity ~0.15 и текст цветом категории.
- Элемент списка упражнения: слева изображение 50×50pt (скругление 8pt, плейсхолдер «photo» при отсутствии), справа — название (17pt), подзаголовок — категория (бейдж). Справа иконка «chevron.right» вторичная. Отступы вертикальные 8pt.
- Добавление/редактирование/удаление из контекстного меню или кнопок.

6.8) «Добавить/Редактировать упражнение» (Add/Edit Exercise)
- Поля: название (TextField, лимит 100 символов), категория (Picker/Segmented), изображение (кнопки «photo»/«camera»), предпросмотр.
- Кнопка «Сохранить» (primary). Ошибки валидации — под полями (вторичный текст красного тона Danger).

6.9) «Детали упражнения» (ExerciseDetail)
- Шапка: изображение упражнения (соотношение сторон сохранять; скругление 12pt; если нет — большой плейсхолдер «photo» центрирован).
- Инфо: категория (бейдж), дата создания, общий объём/лучший результат.
- История: список последних сессий, каждый элемент — дата + параметры (подходы/повторения/вес), возможно график мини-линии справа (опционально, см. Прогресс).

6.10) «Прогресс» (ProgressStats)
- Верхний блок KPI: 3–4 карточки по 1/2 ширины в две строки или горизонтальный скролл. Каждая карточка: крупное число (H2), подпись 12–13pt, иконка справа вторичная.
- Графики: простой линейный график прогресса по датам (формат оси X «dd.MM»), точки с небольшим радиусом, линия Primary 2pt. Оси минимально выражены (тонкие деления).
- Распределение по категориям: столбцы/диаграмма, цвета равны цветам категорий, подписи снизу краткие.
- Топ упражнений по 1RM: таблица или список с иконкой «scalemass», значение справа выровнено по правому краю.
- Личные рекорды (1RM): карточки с названием упражнения и числом, бейдж «PR» (капсула, Primary 10–15%).
- Тепловая карта активности: сетка календаря, ячейки 20–24pt, интенсивность заливки по активности недели/дня, закругление 4pt.

6.11) «Достижения» (Achievements)
- Список бейджей: карточки или плитки 2 по ряду на больших экранах; на малых — по одной.
- Каждый бейдж: иконка SF (из набора: figure.walk, dumbbell.fill, trophy.fill, medal.fill, star.fill, flame.fill, calendar, chart.bar.fill, heart.* и т.д.) 28–36pt, заголовок (15–17pt), подзаголовок/прогресс (14–15pt). Цвет бейджа — зависит от типа/уровня: начальный — gray/orange, средний — blue/indigo, высший — yellow, специальные PR/кардио — red/purple/pink.
- Прогресс: индикатор выполнения (progress bar тонкий 4–6pt), подпись справа «X/Y» или «N%».
- Разблокированные — полная насыщенность цвета; заблокированные — 50% opacity и/или десатурация, замочек вторичный.

6.12) «Профиль» (Profile)
- Шапка: аватар (80–96pt круг), имя, email, краткая статистика (тренировки, упражнения, подходы, общий вес) в строку с разделителями.
- Раздел «Настройки»: пункт «Тема» с переходом к ThemePicker (см. ниже), «Изменить профиль», «Сменить пароль», «Выйти» (красный Danger для выхода).

6.13) «Экран приветствия/регистрации» (Welcome/Register)
- Welcome: логотип/иконка (dumbbell.fill) крупная, заголовок, подпись, кнопки «Регистрация» (primary), «Войти» (secondary/plain).
- Register/Login: поля email/username/password, подсказки и ошибки валидации. Кнопка «Продолжить». Ссылки «Забыли пароль?» — вторичный.

6.14) «Таймер отдыха» (RestTimer)
- Крупный циферблат/время (H1), ниже кнопки «Старт/Пауза/Сброс». Большая primary кнопка, вторичные — outline. Анимация прогресс-окружности 0.3s ease-in-out.

7) Иконки (SF Symbols) — справочные ключи
- Главная: house.fill
- Тренировки: dumbbell.fill
- Упражнения: list.bullet
- Прогресс: chart.line.uptrend.xyaxis
- Достижения: trophy.fill
- Таймер: timer
- Профиль: person.circle
- Общие: plus, pencil, trash, photo, camera, calendar, clock, scalemass, repeat, arrow.clockwise, checkmark, gear, sun.max.fill, moon.fill

8) Состояния и интеракции
- Наводка (iOS): использовать системные эффекты выделения рядов списка.
- Нажатие: лёгкое уменьшение масштаба (0.98) и снижение непрозрачности на 10–15% для кнопок/карточек.
- Disabled: 50% opacity и блокировка жестов.
- Swipe-to-delete: кнопка с красным фоном Danger, иконка «trash», белый текст.
- Pull-to-refresh: системный индикатор.
- Хаптика: light impact при успехе/сохранении, warning при ошибках.

9) Темы (ThemeManager)
- Режимы: system/light/dark.
- Светлая тема: фоны #FFFFFF, карточки #FFFFFF, текст #111111/#6C6C70, разделители системные. Primary/semantic — как указано выше.
- Тёмная тема: фоны #000000/#0C0C0D, карточки #1C1C1E, текст #FFFFFF/#9A9AA0, разделители системные. Цветовые акценты сохраняют относительную насыщенность.
- Переключатель темы (ThemePicker): вертикальный список вариантов, каждая строка — HStack с иконкой слева (gear/sun.max.fill/moon.fill), названием «Системная/Светлая/Тёмная» и чекмарком справа у текущей темы. Активная — фон Primary с opacity 0.1, скругление 8pt.

10) Форматы дат (локаль ru_RU)
- full: «EEEE, d MMMM yyyy 'в' HH:mm»
- medium: «d MMMM yyyy»
- short: «dd.MM.yyyy»
- time: «HH:mm»
- chart: «dd.MM»

11) Ограничения ввода
- Подходы: 0–20
- Повторения: 0–100
- Вес: 0.0–500.0 (кг, шаг 0.5/1.0 по контексту)
- Длина заметок: до 500 символов (индикатор остатка при 450+).
- Длина названия упражнения: до 100 символов.

12) Диалоги и уведомления
- Удаление тренировок/упражнений — системный alert с подтверждением: заголовок, пояснение, кнопки «Удалить» (красная, destructive) и «Отмена».
- Ошибки сохранения — баннер/алерт с текстом проблемы, без технических деталей для пользователя.
- Успешное сохранение — краткая хаптика и/или мини-баннер «Сохранено». Кнопка закрыть — «×»/swipe.

13) Графики и визуализация
- Линия тренда: толщина 2pt, цвет Primary; точки — радиус 3–4pt, вторичный обвод.
- Столбцы категорий: ширина 16–20pt, скругление 4pt, отступ между столбцами 8pt. Легенда снизу или сбоку.
- Теплокарта: шкала от светлого варианта категории/Primary (минимум) до насыщенного (максимум), 4–5 градаций.

14) Доступность (Accessibility)
- Контрастность кнопок/текста — соответствовать WCAG AA для системных тем.
- Размеры кликабельных зон — минимум 44×44pt.
- VoiceOver: семантические метки на кнопках (например, «Добавить тренировку», «Удалить упражнение»).
- Динамический шрифт: поддержка системного изменения размеров текста без поломки макета (компоненты растягиваются, переносы без обрезки).

15) Анимации
- Переходы между экранами — системные iOS.
- Микроанимации: 0.2–0.3s ease-in-out на нажатиях/появлениях карточек, прогресс в таймере — плавный.
- Списки: при добавлении элементов — анимация вставки снизу вверх, при удалении — сдвиг влево и растворение.

16) Порядок и иерархия важности
- Первичные действия визуально выражены (Primary), вторичные — outline/plain.
- На информационных экранах (Прогресс/Достижения) основная площадь — под визуализации/бейджи, списки имеют дыхание (отступы 16pt между секциями).

17) Состояния загрузки
- Индикаторы активности — системные ProgressView. Карточки в скелет-заглушке: прямоугольники с скруглением 8–12pt, анимированная заливка (минимальная), высота строк как в обычном состоянии.

18) Плейсхолдеры изображений
- Упражнения: SF «photo» центрировано, цвет вторичный, фон — system background, скругление 8pt. Размеры: список 50×50pt, деталка — ширина контейнера, высота по содержимому с сохранением пропорций.

19) Локализация
- Все тексты интерфейса — на русском. Форматы дат и чисел — ru_RU. Единицы веса — «кг».

20) Таблица соответствия ключевым экранам и иконкам таббара (строго)
- Главная — house.fill — «Главная»
- Тренировки — dumbbell.fill — «Тренировки»
- Упражнения — list.bullet — «Упражнения»
- Прогресс — chart.line.uptrend.xyaxis — «Прогресс»
- Достижения — trophy.fill — «Достижения»
- Таймер — timer — «Таймер»
- Профиль — person.circle — «Профиль»

21) Примерные карточки метрик (канонический вид)
- Внутренние отступы: 16pt. Заголовок секции (H3) — сверху 0pt от верхней границы карточки, 8pt до контента.
- KPI-ячейка: число (H2), под ним подпись 12–13pt вторичная. Раскладка 2 колонки в строке с gap 16pt. Скругление карточки — 12pt, фон — системный.

22) Стандарты списков
- Ячейка списка: вертикальные отступы 12–14pt, горизонтальные 16pt. Разделитель — системный. Ячейки-карточки вместо разделителя имеют между собой 8pt и собственную тень/фон.
- Аватары/иконки слева выровнены по вертикали с первой строкой текста. Вторая строка — максимум 2 строки, затем …
- Disclosure индикатор («chevron.right») вторичный, 16–18pt.

23) Формы и валидация
- Ошибки под полем 12–13pt, цвет Danger. Поле в ошибке — бордер Danger 1pt, фон с красной заливкой 8–10%.
- Обязательные поля имеют индикатор «*» вторичного цвета в плейсхолдере/лейбле.

24) Экспорт/поделиться (если доступно)
- Кнопка «Экспорт данных» — вторичная/outline. Диалог выбора формата. Индикатор прогресса и завершения (баннер «Экспорт завершён»).

25) Производные состояния для категорий
- Цвет текста на фоне категории: всегда читабельный (обычно тёмный текст на светлом фоне; если тёмная категория — переключить на белый).

26) Примеры текстов
- Кнопки: «Сохранить», «Добавить», «Редактировать», «Удалить», «Отмена», «Применить», «Выйти», «Продолжить», «Начать», «Пауза», «Сброс».
- Плейсхолдеры: «Название упражнения», «Заметки к тренировке», «Поиск», «E-mail», «Пароль».

27) Поведение таббара
- Смена вкладки сохраняет состояние списков и скролл-позицию.
- Повторный тап по активной вкладке — прокрутка к началу списка (если применимо).

28) Размеры изображений и иконок
- Список упражнений: 50×50pt (малый 40×40 допустим для компактных групп).
- Иконки в кнопках: 20–24pt.
- Аватар профиля: 80–96pt.

29) Итоговые требования к идентичности
- Совпадение цветов, отступов, типографики, иконок, порядка вкладок и названий.
- Совпадение состояний элементов (normal/pressed/disabled), анимаций и хаптики.
- Совпадение форматов даты/локали, единиц измерения (кг).
- Совпадение структуры каждого экрана, как описано выше.

Эта спецификация должна быть использована как прямой промпт для дизайнера/нейромодели или как чеклист для ручной отрисовки макетов, с целью добиться 1:1 визуального и поведенческого соответствия приложению GymLog.
